srm664

发表于 2015-08-02   |   分类于 acm   |  

#Easy

####题意：

给定$x, y(0 \le x,y \le 10^9)$，要对这两个数做$k(1 \le k \le 2 \times 10^9)$次操作。每次操作为：若$x \le y$ 则$(x,y)$变为 $(2 \times x, y - x)$，否则变为 $(x - y, 2 \times y)$。问经过$k$次操作后，$min(x,y)$是多少 ?

####题解：

设$S = x + y$，若$y < x$ 则经过一次操作后$y变为2 \times y$，否则变为$y - (S - y) = 2 \times y - S$，所以不管$y$的取值如何，经过一次操作后$y$变为$y \mod S$。

结论：经过$k$次操作后，$y$变为$y \times 2^k \mod S$。

复杂度： $O(logk)$

#Medium

####题意：

给定一棵有$n$个节点的树，第$i(0 \le i \le n)$个节点有$\frac{1}{i + 1}$的概率被选中。设被选中的点形成了$k$个连通块，第$i$个连通块中为$si$，问$\sum |si|^2$的期望乘上$n!$是多少。

####题解：

考虑期望的线性性，我们只需要算出每个连通块对答案的贡献即可。$E(|si|^2) = E(\sum_{u,v \in si} f(u,v))$，其中$f(u,v)$代表$u和v是否连通$。

现在问题就转化为求：$\sum_{1 \le u,v \le n} E(f(u,v))$

$E(f(u,v)) = \prod_{p \in S(u,v)} \frac{1}{p+1}$，其中 $S(u,v)$是$u,v$路径上点的集合。

这个问题可以利用dfs在$O(n)$时间内解决。

时间复杂度：$O(n)$

题目链接

原题解链接

####题意：

在一个$m(1 \le m \le 11)$维空间内有$n(1 \le n \le 300000)$个点，且这$n$个点每一维的坐标都只能为$0、1、2$，点与点之间的距离为曼哈顿距离。对于每个$k(0 \le k \le 2 \times m)$，问有多少对点的距离恰好为$k$。

####题解：

将每个点用一个长度为$m$的字符串表示(字符集为$0$、$1$、$2$)。设$dp[S1][S2][j]$代表：所有前缀为$S1$的点中，和字符串$S1+S2(|S1 + S2| = m)$的距离为$j$的有多少点。

当 $S2$为空时：$dp[S1][S2][0] = sum(S1)，dp[S1][S2][j] = 0$，其中$j > 0$，$sum(S1)为S1的个数。$

当 $S2$不为空时：$dp[S1 + t][S2][j] = \sum_{l = 0}^2 dp[S1 + l][S2][j-abs(t-l)]$

dp总状态数为：$O(3 ^ n \times n m)$，每次转移的代价为$O(1)$，所以总复杂度为：$O(3^n \times nm)$。

####总结：

个人感觉这是一道相当不错的$dp$，之前没有想到把点看成串来定义$dp$的状态。

hdu5329

发表于 2015-07-31   |   分类于 acm   |  

题目链接

####题意：

给定一个有$n$个点$n$条边的连通图，现在要把图划分成若干个拥有$k$个节点的联通子图，问存在多少个$k(1 \le k \le n)$使得其满足条件。

####题解：

题目给的图相当于是一个环带着若干个外向树。

现在考虑一个简单的问题：原图是树且$k$已经确定，判断是否可以划分。该问题可以通过贪心来解决，从叶子开始往上依次处理。如果某个子树的节点个数为$k$，那么把该子树一定当作一个块从树中删除，否则该子树中的任意一个子树都不能构成大小为$k$的连通块。如果节点个数大于$k$，由于该子树是由若干个节点小于$k$的连通块通过根节点连接起来的，所以可以直接返回不合法。这样就可以在$O(n)$时间内解决该问题。

显然，当$k$是$n$的约数时才有可能合法，所以我们可以通过枚举$n$的所有约数在$O(f(n) \times n)$的时间内解决树的问题。($f(n)$为$n$的约数个数)

对于环加外向树的情况，做法也是类似的。对于每棵外向树，我们先把每棵外向树都做一遍上述的贪心处理，然后每棵外向树$i$都剩下了若干个节点，设$fi$为节点个数。现在问题就转化为：给定一个环，环上节点的权值为$fi$，可以把相邻的两个节点合并为一个，新节点的权值为旧节点的权值的和，问：是否可以通过若干次操作，使得环上的每个点权值都为$k$。对于这个问题，首先有个$n^2$的做法：在环上枚举一个位置，将其断成一条链，剩下的就直接$O(n)$判定即可。事实上，我们只需要$O(n)$预处理出从每个位置开始最多能形成的大小为$k$的节点数量，就可以$O(1)$判定是否合法了。

总时间复杂度为：$O(f(n) \times n)$

hdu5314(树分治)

发表于 2015-07-30   |   分类于 acm   |  

题目链接

####题意：

给定一棵点带权的树，问有多少对节点$(u,v)$满足从$u$到$v$的简单路径上点权最大值和最小值的差不超过D，$(u,v)$是有序对且不相等。

####题解：

考虑点分治，假设不经过分治中心的路径都已经处理完毕，现在需要处理经过分治中心的路径。设$Ai$为第$i$个点到分治中心路径上的点权最小值，$Bi$为第$i$个点到分治中心路径上的点权最大值。将该分治阶段的所有点按$Ai$从大到小排序，然后依次枚举，第$i$个点对答案的贡献是 $\sum_{j < i,Bj <= Ai + D}1$ ，这部分可以用树状数组实现。但是有可能出现节点$(u,v)$都属于同一棵子树，我们需要剔除这样的答案。对于这部分的答案，只需要对于每一棵子树，只存该子树中节点的$Ai$和$Bi$然后做一遍上述操作就可以得到。

总时间复杂度为$O(nlog^2 n)$

hdu5322

发表于 2015-07-29   |   分类于 acm   |  

题目链接

####题意：

一个$n$的排列${a1,a2,…,an}$对应一张图$G$，若$(i,j)$满足$ai < aj, i < j$且不存在$k$使得$ai < ak，i < k < j$，则节点$i$和节点$j$之间连一条无向边。设$G$中一共有$k$个联通块，第$i$个联通块有$Ai$个节点，则$\prod Ai$为排列${a1,a2,…,an}$的权值，对于$n$的所有排列，权值总和为多少。

####题解：

假设$n$放在排列的第$i$个位置，则前$i$个节点形成一个大小为$i$的联通块，后$n-i$个节点成为大小为$n-i$的子问题。设$F[i]$为i的答案， 则

$F[n] = \sum_{i=1}^n \binom{n-1}{i-1} \times F[n - i] \times (i-1)! \times i^2 $

$= (n-1)! \times \sum_{i=0}^{n-1} (\frac{F[i]}{i!} \times (n^2 + i^2-2 \times n \times i))$

然后就可以在$O(n)$时间内算出$F[1],F[2],…,F[n]$了。

uva12436(线段树)

发表于 2015-07-27   |   分类于 acm   |  

题目链接

题目大意：有一个长度为$250000$的序列$a$，刚开始序列中的元素均为0。现在有四种操作：

  • $A\,l\,r$ : $a[i] = a[i] + i - l + 1$ ($l \le i \le r$)
  • $B\,l\,r$ : $a[i] = a[i] + r - i + 1$ ($l \le i \le r$)
  • $C\,l\,r \, x$ : $a[i] = x$ ($l \le i \le r$)
  • $S\,l\,r$ : 返回$\sum_{i=l}^r a[i]$

主要思路：维护两棵线段$tree1$和$tree2$，$tree1$维护的是$a[x]$的和，$tree2$维护的是$a[x]*x$的和。则四种操作可以转化为：

  • $A \, l \, r$ : $tree2$区间[l,r]加1，$tree1$区间[l,r]减$l-1$
  • $B \, l \, r$ : $tree2$区间[l,r]减1，$tree1$区间[l,r]加$r+1$
  • $C \, l \, r \, x$ : $tree2$区间[l,r]赋值为0，$tree1$区间[l,r]赋值为$x$
  • $S \, l \, r$ : 就是普通的线段树求和操作。

这四种操作都可以在 $O(log(250000))$ 时间内完成，所以总复杂度为 $O(T*log(250000))$。

hihocoder1193(线段树合并)

发表于 2015-07-27   |   分类于 acm   |  

题目链接

主要思路： 设$ans[v]$代表节点$v$的答案，$T(v,p) = max(\sum_{i = 1}^k ans[ui])$ {$ui$是权值不大于$p$的$v$的后代，且$ui,uj(1 \le i < j \le k)$互不为祖孙关系}，则

$ans[v] = T(v,value[v]) + 1$

将$T(v,p)$用一棵以$p$为下标的权值线段树$treev$来代表，另设$tree1v$代表在$treev$中加入了节点$v$的答案。 由于在处理节点$v$的时候，$v$的子树之间互不相关，所以可以通过合并子树的$tree1u$来得到$treev$，以及得到$ans[v]$。显然可以用$ans[v]$更新$treev$的值来得到$tree1v$。线段树中的权值是单调递增的，且$tree(v,value[v])$的值只比$ans[v]$少1，所以通过在区间加上1来更新$treev$。

主要涉及到的操作为：线段树合并，线段树单点查询，线段树区间加1。
